The substrate · Agent SDK

The primitives everything is built on

🎤 Anthropic says

  • The Agent SDK runtime: a core agent loop (Claude decides the next action → tool call → result), tools (Read/Edit/Bash/WebSearch/Task/MCP), subagent loops spawned by the Task tool, a permission gate, hooks (PreToolUse, Stop…), project context (CLAUDE.md, Skills, slash commands), and resumable sessions.
  • Andrew’s framing: “you use this framework to build your own harness.” The primitives ship; the harness is yours.
loops.md § “Two approaches: Model updates vs. Harness evolution”, ~4:51

⚖️ Specflow vs Anthropic Context

  • Specflow is a harness built on these primitives, not a rival to them. specflow run is the agent-supervised controller; our adapters shell out to claude -p / codex exec.
  • Where the SDK exposes Permissions + Hooks + Sessions, Specflow adds a contract layer on top: run-contract.yaml (state), ledger.jsonl (durable memory), and mechanical gates (verification).

The problem

Three reasons agents lose the plot

🎤 Anthropic says

  • Context — finite window, amnesia between sessions, coherence degrades as it fills, and “context anxiety” makes it wrap up early.
  • Planning — one-shots the whole project, or stubs half a feature and calls it done, or runs out of context mid-feature.
  • Verification — tests shallowly (curl, not click), stubs features, and rates mediocre work as good. Uncorrected, all three compound.
loops.md § “Challenges: Context, Planning, and Judgment”, ~2:29

⚖️ Specflow vs Anthropic Aligned

  • These map 1:1 onto Specflow’s three answers. Context → durable run-contract + ledger (state survives the window).
  • Planning → spec-build emits the plan as durable artifacts; feature-build executes one rail at a time.
  • Verification → mechanical gates + provenance. The whole thesis: put state and judgment on disk, not in the model.

Interlude · prior art

The Ralph technique

🎤 Anthropic says

  • Geoffrey Huntley, Jul 2025: while :; do cat PROMPT.md | claude-code; done. Infinite loop, one task per iteration, a fix_plan.md the agent updates itself.
  • “Deterministically bad in an undeterministic world” — better to fail predictably. Claude Code’s version is a Stop hook + iteration guard + safe word (the same loop, no shell wrapper).
loops.md § “The Ralph loop technique”, ~7:56

⚖️ Specflow vs Anthropic Specflow ahead

  • specflow run --until-terminal is a Ralph-style bounded loop — but with the half Ralph is missing.
  • Ralph has a fixed plan.md and nobody argues with the main loop. Specflow inserts a run-contract (goal/state/verifier/stop) and a mechanical gate between iterations.
  • Verdict: Specflow = Ralph loop + a contract + a critic. Keep the loop; reject “fixed plan, no judge.”

The money chart

Every release, agents run longer unattended

🎤 Anthropic says

⚖️ Specflow vs Anthropic Context

  • The growing horizon is why the fable packet exists: longer autonomy = larger unsupported changes = more need for state, isolation, and independent verification (fable PRD problem statement).
  • Verdict: longer unattended runs are the argument FOR contracted loops, not against them. Specflow’s value grows with the horizon.

The core pattern

Separate the builder from the judge

🎤 Anthropic says

  • Generator builds → Evaluator grades → critique back. Inspired by GANs: split the context windows, system prompts, and jobs entirely.
  • Crucially, the evaluator uses Playwright to navigate the live page before scoring — it doesn’t read diffs, it drives the product.
loops.md § “State-of-the-art harness patterns”, ~18:27

⚖️ Specflow vs Anthropic Specflow gap

  • This is our trust model: provider output is never a gate result (I-ADAPTER-001). #84 VERIFIER-01 makes the split explicit — verifier policy separate from maker policy.
  • The gap: their evaluator drives the running app; our verifiers mostly inspect artifacts.
  • Verdict: aligned on the split; #84 must adopt the “evaluator uses the app” half, not just plan-review.

The key insight

Tuning a skeptic is tractable; self-criticism isn’t

🎤 Anthropic says

  • “Tuning a standalone evaluator to be skeptical is tractable. Making a generator self-critical is not.”
  • Same as humans — easy to critique a meal, hard to cook it. Exploit the gap between an LLM as critic and the same LLM as generator.
loops.md, ~19:50

⚖️ Specflow vs Anthropic Specflow ahead

  • We go one better: where it matters our critic isn’t an LLM at all — it’s a deterministic script (verify-seams/verify-adr/provenance). A script can’t be flattered.
  • Where we do add an LLM critic (#84 external reviewer, #88 vision) we inherit their rule: harsh rubric, artifact + rubric only, never the maker’s trace.
  • Verdict: mechanical critic > tuned-skeptical LLM critic. Keep LLM critics subordinate to the script.

Grading taste

Making subjective quality gradable

🎤 Anthropic says

  • You can grade taste if you write it down. Four criteria: Design (coherent whole?), Originality (custom vs library defaults), Craft (typography/spacing/contrast), Functionality (tasks without guessing).
  • Weighted toward Design & Originality (4.6 already nails Craft & Functionality). Kills purple-gradient “AI slop.”
loops.md § “Evaluating subjective output with rubrics”, ~21:31

⚖️ Specflow vs Anthropic Aligned

  • Same philosophy as our contracts: write the standard down as checkable criteria. Their weighted rubric is exactly what #84’s verifier rubric and #88’s vision rubric should be — granular and weighted, not vague.
  • Their finding “vague criteria → the generator shrugs; 27 granular criteria → it fixes the exact line” is the Specflow thesis restated.
  • Verdict: strong validation of contracts; port the weighted-rubric shape into #84/#88.

🎤 Anthropic says

⚖️ Specflow vs Anthropic Tension

  • Our loop advances monotonically (stage → next on pass; fail → blocked). We have no “discard the slice and restart from scratch” primitive — that’s a human decision, by design (auditability).

  • Verdict: tension — their restart-happy loop vs our monotonic, ledgered loop. Ours trades exploration for auditability, correct for our domain.

Scaling up

Full-stack: three agents

🎤 Anthropic says

  • Add a Planner: one-line prompt → full spec. Generator builds one sprint at a time. Evaluator = Playwright + rubric, handoff / pass-fail.
  • Squint and it’s just PM / IC / QA — “we didn’t invent this; we gave each role its own context window.”
loops.md § “Introducing the ‘Planner’ role”, ~23:45

⚖️ Specflow vs Anthropic Aligned

  • This is literally our loop set: Planner ≈ spec-build, Generator ≈ feature-build rails, Evaluator ≈ gates + #84 verifier.
  • We reached the same PM/IC/QA split — but we persist each role as contracts and issues rather than ephemeral agent roles.
  • Verdict: same three-role architecture; Specflow makes each role a durable, CI-enforceable artifact.

The key innovation Ralph never had

Sprint contracts — negotiate “done”

🎤 Anthropic says

  • Before any code, generator and evaluator negotiate what “done” means for this chunk, iterating via files until they agree.
  • The spec is intentionally high-level; the contract bridges user stories → testable behaviors; the evaluator grades against the contract, not the original spec.
loops.md § “The generator-evaluator contract”, ~24:56 / ~26:33

⚖️ Specflow vs Anthropic Specflow gap

  • This is the sub-stage #84 is missing. Our spec-build authors the contract up front; theirs is negotiated at build time by maker + verifier.
  • Recommendation: keep ticket ACs as the high-level spec, then add a maker↔verifier negotiation that derives the granular test assertions to disk before building.
  • Verdict: adopt the negotiation into #84 — the one primitive our loop lacks. (But see slide 19: they later removed even this.)

Honest admission

“Out of the box, Claude is a poor QA agent”

🎤 Anthropic says

  • “It would identify legitimate issues, then talk itself into deciding they weren’t a big deal and approve the work anyway.”
  • It took exorbitant rubric-tuning to fix — the same sycophancy that hits any LLM-as-judge.
loops.md § “Specificity in contracts and debugging traces”, ~32:40

⚖️ Specflow vs Anthropic Specflow ahead

  • This is the exact failure our mechanical gate exists to prevent — a script never “talks itself into” a pass.
  • It’s also the warning label for #84/#88: any LLM verifier we add starts here and must be tuned harsh + kept subordinate to the gate.
  • Verdict: vindicates “no self-grading” and sets the bar for any LLM critic we introduce.

The primary debugging loop

Read the traces. Repeat.

🎤 Anthropic says

  • “Read the evaluator’s logs. Find where its judgment diverges from yours. Update the prompt. Repeat.”
  • Reading traces by hand is the primary debugging loop; piping to another agent is only a first pass. Observability for long runs is unsolved green-field software.
loops.md § “Specificity…”, ~33:26; Q&A ~1:01:25

⚖️ Specflow vs Anthropic Specflow gap

  • We produce the ideal artifact — ledger.jsonl + transcript_path — and ship zero tooling to read it. No fable ticket covers this.
  • Proposal: candidate #90 — specflow run trace / a maker-vs-gate divergence report.
  • Verdict: clear gap; the talk’s #1 practice has no Specflow tooling yet.

The end state

The simplified harness, end to end

🎤 Anthropic says

  • Human → Planner (prompt→spec, once, ~5 min) → Generator (one continuous build, no resets, 2+ hrs) ↔ Evaluator (Playwright + rubric, at end, 2–3 passes).
  • The filesystem is shared state: spec.md, app/ + git, findings.md. “Three agents, zero shared context — all coordination through files on disk.” Removed on 4.6: context resets, sprint decomposition, per-sprint eval loop, and sprint contracts.
loops.md, ~36:08

⚖️ Specflow vs Anthropic Aligned

  • The end state is pure Specflow philosophy — file system as shared state, not context. Their spec.md/findings.md = our run-contract.yaml/ledger.jsonl/spec dir.
  • Sharpening slide 14: they removed the per-sprint contract negotiation too — so that step is itself 4.5-era scaffold. The durable core = planner + maker/judge split + files-on-disk.
  • Verdict: our architecture already matches their simplified end state. Treat negotiation as optional; keep the file-based handoff as bedrock.

The take-a-photo slide

Five takeaways

🎤 Anthropic says

  • 1. Self-evaluation is a trap → use an adversarial evaluator.
  • 2. Compaction doesn’t cure coherence drift → structured handoffs do.
  • 3. Make subjective quality gradable with rubrics the model can apply.
  • 4. Read the traces — they’re your primary debugging loop.
  • 5. Delete scaffolding when the model catches up — the frontier moves.
loops.md § “Key takeaways for long-running agents”, ~38:56

⚖️ Specflow vs Anthropic Context

  • 1 → Ahead (mechanical gate; #84 formalizes the split).
  • 2 → Aligned (run-contract + ledger = structured handoff).
  • 3 → Aligned / core (contracts; port the weighted rubric to #84/#88).
  • 4 → Gap (no trace tooling — propose #90).
  • 5 → Watch (keep #83/#89 thin; durable primitives heavy).

Closing

Same architecture, opposite directions

🎤 Anthropic says

  • Anthropic arrived here from greenfield one-shot demos ($200, 6 hours, a retro game maker) — optimising for taste and autonomy with no one auditing why.
  • Specflow arrives from brownfield, auditable, no-ticket-no-code delivery — optimising for traceability and CI enforcement. We converged on the same planner / maker / judge + files-on-disk shape.
